Privacy Policy – Providence Find Care (Demo)

Last updated:

This is a demonstration application (“Demo”) that helps users choose an appropriate care venue (ER, urgent, primary, or virtual), find nearby Providence locations, view mock availability, and generate a mock booking deep link. It is not intended for real patient care, and it does not diagnose conditions.

What data we process

  • Symptoms and preferences you type are sent to our backend solely to fulfill your request (triage, location search, availability, mock booking).
  • No PHI is stored. We do not persist user-entered symptoms, ages, or booking selections in any database.
  • Patient context token (if provided) is treated as an opaque, short‑lived string and is not used to identify you.
  • Standard service logs (e.g., IP address, user agent, timestamps, request path/status) may be recorded by our hosting provider for reliability and security. These logs are not linked to any medical information you provide.

Third‑party services

  • Hosting: Azure App Service (Microsoft) runs the containerized backend.
  • CI/CD & container registry: GitHub Actions and GHCR are used to build and distribute the demo container image.
  • Maps: A simple Google Maps link (https://maps.google.com/?q=lat,lon) is displayed in an iframe; no map API keys are used.

Retention

We do not intentionally store user‑entered content. Hosting platform access logs may be retained for up to 30 days for operational purposes and then discarded.

Security

Transport is encrypted via HTTPS. This demo avoids persistent identifiers and does not store medical content server‑side. As a demonstration, it has not undergone formal security certification.

Your choices

  • Do not enter real personal or medical details; use test data only.
  • You may close your browser or stop using the demo at any time.

Contact

For questions about this demo or this policy, please contact the repository owner or open an issue on the project’s GitHub repository.
